greentea-client

Note: This copy of greentea-client is used only with Mbed CLI 1. Mbed CLI 2 will use the standalone greentea-client, automatically fetched by CMake in theextern/ directory". <tt>greentea-client</tt> is a client library for <a href="https://github.com/ARMmbed/mbed-os-tools/tree/master/packages/mbed-greentea" >the Greentea test tool</a> when used in an <a href="https://os.mbed.com/" >Mbed OS</a> project. This package implements the client side of the key-value protocol used for communication between the device under test (DUT) and the host. The <tt>Greentea</tt> tool implements the protocol's host behavior. The test suite has a beginning and an end (like the <tt>main()</tt> function would. The test suite may pass, fail or be in an error state (for example, if the test suite times out or there was a serial port connection problem). @subsubsection autotoc_md682 Test case A test case is contained within a test suite. You can have multiple test cases within a test suite. <tt>unity</tt> assert macros are used to run the checks during the test case. Your test cases may pass, fail or be in an error state. @subsubsection autotoc_md683 Key-value protocol The key-value protocol is a protocol specific to the <tt>Greentea</tt> test tool. It is used to send messages (events) between the DUT and the host. Each message consists of a <em>key</em> and <em>value</em> pair. A message is surrounded by double curly braces. The key and value are separated by a semicolon (<tt>;</tt>). For example, the <tt>{{timeout;120}}}</tt> string is a key-value message where the key <tt>timeout</tt> has the value <tt>120</tt>. Both <tt>greentea-client</tt> and <tt>Greentea</tt> understand this format and can detect key-value messages in a data stream. @subsection autotoc_md684 Test example @icode{c++} #include "mbed.h" #include "greentea-client/test_env.h" #include "utest/utest.h" #include "unity/unity.h" void test_case_1_func() { // Test case #1 body // Here you can run your test cases and assertions TEST_ASSERT_TRUE(true); TEST_ASSERT_FALSE(false); } void test_case_2_func() { // Test case #2 body // Here you can run your test cases and assertions TEST_ASSERT_TRUE(true); TEST_ASSERT_FALSE(false); } const Case cases[] = { Case("Test case #1 name", test_case_1_func), Case("Test case #1 name", test_case_2_func) }; status_t greentea_setup(const size_t number_of_cases) { GREENTEA_SETUP(5, "default_auto"); return greentea_test_setup_handler(number_of_cases); }
