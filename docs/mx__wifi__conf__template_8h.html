◆ MX_WIFI_MIN_TX_HEADER_SIZE

#define MX_WIFI_MIN_TX_HEADER_SIZE   (28)

For the TX buffer, by default no-copy feature is enabled, meaning that the IP buffer are used in the whole process and should come with available room in front of payload to accommodate transport header buffer. This is managed in interface between driver and IP stack For LwIP "PBUF_LINK_ENCAPSULATION_HLEN" must be defined as > MX_WIFI_MIN_TX_HEADER_SIZE, see net_mx_wifi/c file for implementation.
